# 小红书广告投放分析工具

一站式处理小红书图文链接的数据下载、证据整理与广告投放潜力分析，面向转转内部研究原型场景。

## 安装部署

### 系统要求

- Python 3.10+
- Node.js 18+
- Playwright 浏览器环境
- 可用的小红书登录态缓存
- 足够的本地磁盘空间，用于保存图片、JSON、评论与分析报告

### 一键安装思路

本 skill 的目标不是从零搭建新系统，而是尽量复用两个基座项目：

| 基座 | 作用 | 使用原则 |
|------|------|------|
| `NanmiCoder/MediaCrawler` | 小红书数据下载与整理能力来源 | 优先复用，非必要不改原能力 |
| `ALBEDO-TABAI/video-copy-analyzer` | `SKILL.md` 组织结构、文风、阶段式工作流基座 | 强对齐其主结构，只做任务相关最小修改 |

建议安装步骤：

```bash
# 1. Python 依赖环境
python3 --version
pip3 --version

# 2. Node.js 环境
node --version
npm --version

# 3. Playwright 浏览器环境
python3 -m playwright install

# 4. 准备 MediaCrawler 所需依赖
# 按 MediaCrawler 官方文档完成 uv / python / node / playwright 依赖安装
```

### ⚠️ 部署提醒

- 本 skill 当前定位为内部研究原型，不把当前阶段视为可直接商用交付。
- 小红书数据下载能力应优先复用 `MediaCrawler` 的既有小红书能力，包括但不限于：
  - 图片
  - 文本
  - 指定帖子 ID 爬取
  - 关键词搜索
  - 二级评论
  - 指定创作者主页
  - 登录态缓存
  - IP 代理池
  - 评论词云图
- 当前主闭环围绕“用户上传链接对应的帖子明细抓取与分析”，但实现时不得把上述能力从总设计里丢失。

## 工作流程（5 阶段）

> **重要：你必须严格按照以下 5 个阶段顺序执行，每个阶段完成后再进入下一个阶段。不要跳过任何阶段。**

### 阶段 1: 输入检查与链接提取

**目标**：从用户自然语言输入中提取全部链接，识别小红书链接并完成初步分类。

**执行步骤**：
1. 读取用户本轮输入的完整原始文本。
2. 从混合文本中提取全部 URL。
3. 用 `http://xhslink.com/` 前缀判断是否为有效小红书链接。
4. 记录所有非小红书链接，但不要把它们纳入后续分析。
5. 统计有效小红书链接数量。
6. 若有效链接数量为 `0`，立即返回错误提示并停止后续流程。
7. 若有效链接数量大于 `1`，必须按“每个链接一个独立样本”的方式继续，不允许把多个链接混成一个样本。
8. 注意区分：
   - 用户输入中的任务说明
   - 小红书分享内容中附带的标题文案
   - `复制后打开【小红书】查看笔记！` 之类的噪声文本

**确认成功**：

- 已得到有效小红书链接列表
- 已得到非小红书链接列表
- 已保留原始输入文本供后续落档

---

### 阶段 2: 小红书数据下载与整理

**目标**：围绕每个有效小红书链接，调用小红书数据下载能力，收集足够的原始材料并整理到本地。

**执行步骤**：
1. 为当前任务创建 run 目录。
2. 为每个有效小红书链接创建独立样本目录。
3. 样本目录命名遵循：
   - 展示名：`完整标题 + 作者昵称`
   - 实际落盘名：可做文件系统安全化处理，但必须在 manifest 中保留原始标题与作者昵称
4. 基于 `MediaCrawler` 的小红书能力，围绕每个链接尽量完整下载并整理以下内容：
   - 图文正文文本
   - 图片文件
   - 评论区内容
   - 互动数据
   - 标题、封面、标签、发布时间、作者信息等辅助元数据
   - 评论词云图等衍生资产（如能力可用）
5. 对下载得到的材料按类型分目录整理，保证后续 LLM 分析能稳定读取。
6. 为每个样本生成用于分析与溯源的 manifest / 引用索引。

**必须保留的能力边界**：

| 能力 | 在总设计中的地位 | 当前主闭环中的角色 |
|------|------------------|--------------------|
| 指定帖子 ID 爬取 | 必须保留 | 当前主路径核心 |
| 关键词搜索 | 必须保留 | 非主路径，但不能从总设计中遗漏 |
| 二级评论 | 必须保留 | 当前分析重要补充证据 |
| 指定创作者主页 | 必须保留 | 当前非主路径扩展能力 |
| 登录态缓存 | 必须保留 | 环境稳定性前提 |
| IP 代理池 | 必须保留 | 扩展能力与稳定性能力 |
| 评论词云图 | 必须保留 | 衍生分析资产 |

**确认成功**：

- 每个有效链接都有独立目录
- 每个目录下至少包含可用于分析的文本、图片、评论和可用互动数据
- 原始下载数据已整理为后续可引用、可传递的结构

---

### 阶段 3: Prompt 选择与 run 级资产准备

**目标**：确定使用标准 Prompt 还是 run 专属 Prompt，并为后续分析准备本次 run 的输入资产。

**执行步骤**：
1. 判断用户是否只上传链接，或附带的任务目标是否与标准任务严格等价。
2. 若满足以下任一条件，则使用标准 Prompt：
   - 用户仅上传链接和附带分享标题，并无额外任务要求
   - 用户额外要求与标准任务目标完全相同
3. 若用户提出了不同的额外分析要求，则：
   - 不引用全局 `assets` 中的标准 Prompt
   - 在当前 run 中写入新的专属 Prompt 文件
   - 该 Prompt 仅可被当前 run 使用，绝对不能写回全局 `assets`
4. 将本次实际使用的 Prompt 明确写入 run 目录。
5. 为所有分析材料建立清晰引用锚点，确保所有结论可回溯到具体文件和位置。

**确认成功**：

- 已明确本次 run 的 Prompt 模式：`standard` 或 `run-specific`
- 已把实际使用 Prompt 落到当前 run
- 已建立引用锚点和引用来源索引

---

### 阶段 4: 单篇分析与综合分析

**目标**：基于当前 run 的下载材料和 Prompt，生成单篇分析报告；若样本数大于 1，再生成综合报告。

**执行步骤**：
1. 逐篇读取每个样本的文本、图片、评论、互动数据和辅助材料。
2. 先生成每个样本的独立 Markdown 报告。
3. 若有效链接数为 `1`，则只输出单篇报告，不执行综合报告。
4. 若有效链接数大于 `1`，则在全部单篇报告完成后，再基于本次 run 内全部单篇报告生成综合分析报告。
5. 任何关键判断都必须附带引用标签。

**标准任务提示词**：

```md
# 小红书图文投放潜力分析 Prompt（标准版）

## 目标

针对用户在本次对话中上传或提供的小红书图文链接所对应的已下载内容，完成逐篇深度分析，并输出单篇分析报告与本次会话级综合报告。

分析对象默认包括：正文文本、图片、评论、互动数据（如已提供）。其中重点分析文本、图片、评论三类内容，以及它们之间的关联优势，解释为什么这些表面上风格不同的图文内容都具备优秀广告投放潜力。

除非用户另有说明，默认这些样本已被业务侧初步验证为具备较高 ROI / CVR 的优质小红书图文样本。

## 一、角色定义

你扮演的角色是：

转转公司在小红书平台上的广告投放分析专员。

你的职责不是简单总结内容，而是：

1. 逐篇分析每一个小红书图文链接下载后的内容材料。
2. 深入理解并拆解该篇内容在以下层面的优势与特征：
   - 文本优势 / 特点
   - 图片优势 / 特点
   - 评论优势 / 特点
   - 文本 × 图片 的关联优势
   - 文本 × 评论 的关联优势
   - 图片 × 评论 的关联优势
3. 判断这些元素之间的耦合和相互强化，解释它们如何共同降低理解门槛或决策阻力，并因此带来更强的广告投放潜力。
4. 对本次对话中的全部单篇报告进行跨内容汇总，回答：
   - 为什么这些内容看起来不同，但最终都能取得好数据？
   - 它们之间有哪些共性、结构性规律与可迁移方法？

## 二、执行步骤

### 步骤 1：识别与读取输入内容

对每一篇小红书图文样本，读取并整理以下信息：

- 图文正文文本
- 图片文件
- 评论区内容
- 互动数据（点赞、收藏、评论数、转发等，如已提供）
- 其他辅助信息（标题、封面、标签、发布时间、作者备注等，如已提供）

若本次对话只有 1 个链接，则仅输出单篇分析报告，不执行综合报告。
若本次对话包含 2 个及以上链接，则在全部单篇报告完成后继续输出综合报告。

你必须对每一项内容建立清晰引用锚点，确保所有分析结论都能追溯到明确来源。

### 步骤 2：生成单篇分析报告

对每一篇图文分别进行深度分析，并输出单独 Markdown 报告：

- `图文标题+作者名_分析报告.md`

仅当本次对话中存在多篇图文时，再额外输出：

- `本次总结出的共有特点+时间戳_综合分析报告.md`

综合报告必须基于本次对话内全部单篇报告，而不是仅基于当前轮输入或历史会话。

## 三、单篇分析框架

### 总体要求

- 所有单篇报告开头必须先给出摘要简报
- 语言尽量简洁，优先输出高密度判断，不写空泛长段
- 不要只用连续文字堆叠，应灵活使用表格、层级结构、对照结构、简洁列表
- 每个大部分都应保留自由发挥区，允许补充框架之外但有价值的观察
- 分析重点始终是：为什么有效、如何有效、是否可迁移
- 对 Hook、Emotion、爆点、CTA、社交货币给出 Viral-5D 评分和简要分析
- 给出内容风格标签

### 3.1 内容要素拆解视角

#### A. 文本分析

- 标题与开头的吸引机制
- 正文表达结构：铺垫、展开、重点、结尾
- 文案语气与叙事方式：真实感、种草感、对比感、经验感、情绪感、专业感等
- 关键词与高价值表达提取
- 用户决策相关信息密度：价格、场景、痛点、收益、对比、信任线索等
- 可能促进转化的关键句和关键段落

#### B. 图片分析

- 首图 / 封面的作用：抓眼、兴趣、信任、价值、反差
- 图片整体风格：真实拍摄、生活化、对比型、清单型、教程型、结果展示型等
- 图片信息组织方式：单张承载、组图递进、图文配合、重点标注
- 强化购买 / 转化意图的元素
- 与平台氛围契合的表达方式
- 具体高价值图片页、区域或视觉元素

#### C. 评论分析

- 高频评论主题归纳
- 用户关注点、疑问点、认可点、争议点
- 评论对正文与图片的补强作用
- 评论是否形成社会证明、口碑背书、真实反馈感
- 评论区是否帮助降低用户决策门槛或增强信任

#### D. 互动数据辅助判断

- 点赞 / 收藏 / 评论 / 转发的结构特征
- 互动数据反映出的内容消费模式
- 数据与内容表现的对应关系
- 对高 ROI / 高 CVR 潜力的辅助解释

若数据不足，不要臆测；可以明确指出判断边界。

#### E. 自由补充观察

补充任何无法被上述子项完整涵盖、但对投放判断有价值的观察。

### 3.2 内容关联耦合视角

必须重点分析元素之间的关联，不允许只拆单个元素。

#### A. 文本 × 图片

- 文本是否为图片提供解读框架
- 图片是否将文本卖点具象化、可信化、生活化
- 是否形成更强的说服链路
- 是否形成“看图被吸引、看文被说服”的组合机制

#### B. 文本 × 评论

- 评论是否验证文本核心主张
- 评论是否放大文本中的卖点、痛点、转化点
- 评论是否补上第三方证据或真实用户视角
- 文本是否主动引发评论讨论

#### C. 图片 × 评论

- 评论是否针对某些图片产生集中反馈
- 图片是否激发代入、提问、模仿、求链接、求教程等高转化信号
- 评论是否帮助解释图片为什么有效

#### D. 三者联动机制总结

- 图片负责吸引与感知
- 文本负责解释与说服
- 评论负责验证与补强
- 若形成闭环，请明确说明闭环路径与其意义
- 若未形成完整闭环，也要指出最强的组合关系

### 3.3 投放潜力诊断视角

#### A. 内容转化潜力判断

- 为什么这篇内容有较强 ROI / CVR 潜力
- 它更影响哪个决策阶段：认知 / 兴趣 / 比较 / 信任建立 / 行动转化
- 它更适合哪类投放资产：强种草型 / 强信任型 / 强对比型 / 强痛点解决型 / 强场景代入型 / 强口碑验证型 / 其他

#### B. 广告友好度分析

- 是否具备可投放放大的结构
- 卖点是否明确
- 信息是否密集但不造成理解负担
- 是否易于快速消费和快速判断价值
- 是否具备可拆分、可改写、可复用的广告素材潜力

#### C. 优势归因

明确区分：

- 单独由文本带来的优势
- 单独由图片带来的优势
- 单独由评论带来的优势
- 由文本 / 图片 / 评论关联产生的额外优势

其中最后一类为重点。

#### D. 风险与边界

- 内容效果是否依赖特定品类、人群或表达风格
- 哪些优点难以直接迁移
- 哪些元素可泛化复用，哪些只能参考不能照搬

### 3.4 结构化结论视角

#### A. 单篇内容核心结论

用 3-5 条高度概括的判断总结：

- 这篇图文为什么有效
- 最值得关注的核心机制
- 最强的内容关联优势

#### B. 可复用方法

提炼该篇内容中可迁移到后续广告投放素材的做法：

- 文本写法
- 图片组织方式
- 评论引导方式
- 内容组合结构

#### C. 后续测试建议

给出 2-4 条可执行测试方向：

- 文案结构替换测试
- 首图样式测试
- 评论引导测试
- 图片顺序测试
- 卖点表达密度测试

## 四、引用与证据要求

你必须为报告中的关键判断提供来源引用，禁止无来源支撑的武断结论。

### 4.1 引用标注要求

在正文中使用引用标签，例如：

- `[T1]`：文本引用
- `[I2]`：图片引用
- `[C3]`：评论引用
- `[D1]`：互动数据引用

引用标签必须紧跟在对应分析结论之后。

### 4.2 引用来源写法

在每份报告末尾增加“引用来源”章节，按学术论文参考资料风格列出。每条来源尽可能包含：

- 来源类型（文本 / 图片 / 评论 / 数据）
- 绝对路径
- 文件名
- 所属图文 ID / 链接标识
- 具体位置
- 必要时附简短摘录或描述

示例：

## 引用来源
[T1] 文本｜/abs/path/post_001/content.txt｜文件：content.txt｜位置：标题 + 第2段｜摘录：“……”
[I2] 图片｜/abs/path/post_001/images/03.jpg｜文件：03.jpg｜位置：第3张图，画面上半部价格对比区域
[C3] 评论｜/abs/path/post_001/comments.json｜文件：comments.json｜位置：第12条评论｜摘录：“……”
[D1] 数据｜/abs/path/post_001/metrics.json｜文件：metrics.json｜字段：like_count / save_count

## 五、单篇输出文件

每一篇图文都必须生成一份独立报告：

## 七、综合汇总分析任务

当本次对话中所有单篇图文都完成分析后，你还必须基于本次对话内全部单篇报告继续生成一份综合报告。

综合报告重点回答：

- 为什么这些表面看起来风格不同、内容不同的小红书图文，最终都能拿到好数据？
- 它们之间有哪些稳定共性？
- 哪些共性属于内容层、结构层、文本 / 图片 / 评论联动层？
- 哪些规律最值得沉淀为投放方法论？

## 八、综合汇总分析框架

### 8.1 共性模式总结

- 文本共性
- 图片共性
- 评论共性
- 数据表现与内容结构的共性关系

### 8.2 差异表象 vs 底层一致性

- 表面风格差异
- 底层一致机制
- 相似说服逻辑、信任机制或转化结构

### 8.3 关联机制共性

- 文本与图片如何共同构建说服力
- 评论如何作为转化补强层存在
- 哪些样本具备明显三者联动闭环
- 哪些联动方式最可能解释高 ROI / 高 CVR

### 8.4 投放方法论沉淀

- 可复用内容框架
- 可复用首图策略
- 可复用文案逻辑
- 可复用评论经营思路
- 可复用素材筛选标准

### 8.5 测试优先级建议

- 哪些变量最值得 A/B 测试
- 哪些特征可作为未来筛选高潜图文的判断标准
- 哪些规律更适合不同品类、人群、投放阶段

### 8.6 自由发挥区

允许补充对策略团队有价值的抽象判断、反例提醒、分类建议或方法升级方向。

## 十一、执行原则

1. 逐篇分析必须充分展开，不能只写概括性总结。
2. 必须同时分析单个元素优势与元素之间的关联优势。
3. 关联分析是本任务重点，优先级高于单独元素分析。
4. 所有关键结论必须给出明确引用来源。
5. 综合报告必须建立在本次对话内所有已生成单篇报告之上。
6. 不能因为内容表面风格不同，就停止抽象底层规律。
7. 最终目标不是内容赏析，而是解释其广告投放潜力及可迁移方法论。
```

**确认成功**：

- 每个样本都有单篇分析报告
- 多样本任务已有综合分析报告
- 每个关键结论都有引用来源

**执行步骤**：

**确认成功**：

- 单篇和综合报告均已落盘

## 执行提醒与故障排除

### 有效小红书链接数量为 0

立即返回错误提示，不进入下载、分析和落档阶段。

### 输入里混有标题文案和复制提示语

优先提取真实 URL，不要把分享文案或“复制后打开【小红书】查看笔记！”误识别为链接或任务要求。

### 输入里同时包含小红书链接和其他平台链接

只分析小红书链接，但必须在最终播报中明确列出哪些链接被排除。

### 用户给出了额外分析要求

只要目标与标准任务不严格等价，就必须生成 run 专属 Prompt，且不得写回全局 `assets/`。

### 单个样本缺少某类材料

可以明确指出判断边界，但不得臆测；仍需基于可用材料完成分析。

### 多样本综合报告

综合报告必须建立在本次 run 的全部单篇报告之上，不能只基于当前轮输入，也不能混入历史 run。
