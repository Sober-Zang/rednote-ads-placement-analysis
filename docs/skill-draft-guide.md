## 5. 工作流草案

未来 `SKILL.md` 采用 5 阶段执行法，章节风格仿照 `video-copy-analyzer`：

### 阶段 1：输入检查与链接提取

目标：

- 从自然语言中提取全部 URL
- 识别 `http://xhslink.com/` 链接
- 标记非小红书链接
- 统计有效链接数量是否大于 0

输出：

- `link_manifest.json`
- 被排除链接清单

### 阶段 2：小红书数据采集

目标：

- 围绕每个有效链接获取文本、图片、评论、互动数据和必要辅助信息
- 以 `MediaCrawler` 的小红书能力作为研究型来源

约束：

- MVP 实现闭环优先围绕“指定链接/帖子明细抓取”
- 对关键词搜索、创作者主页、代理池、评论词云等能力，先在实现蓝图中预留位置，不在本阶段假装已落地

输出：

- 单链接目录中的原始内容资产
- `note_manifest.json`

### 阶段 3：Prompt 选择与 run 整理

目标：

- 判定使用标准 Prompt 还是 run 专属 Prompt
- 整理 run 目录和引用锚点

规则：

- 无额外要求：使用标准 Prompt
- 用户目标与标准任务严格等价：使用标准 Prompt
- 其余情况：生成 run 专属 Prompt，只写入本次 run

输出：

- `prompt/used_prompt.md`
- `reference_index.json`
- `run_manifest.json`

### 阶段 4：单篇分析

目标：

- 对每个有效链接生成一份独立分析报告

分析框架：

- 内容要素拆解
- 内容关系与协同机制
- 投放潜力诊断
- 结构化结论
- 引用来源

输出：

- `<图文标题+作者名>_分析报告.md`

### 阶段 5：综合分析与最终播报

目标：

- 当有效链接数大于 1 时，对本次 run 的全部单篇报告做共性总结
- 输出综合分析报告和最终播报

输出：

- `<本次总结出的共有特点+时间戳>_综合分析报告.md`
- 终态播报文本

## 6. 标准 Prompt 与 run 专属 Prompt 的分流

标准 Prompt：

- 存放于未来 skill 的 `assets/` 目录
- 用于标准任务或严格等价任务

run 专属 Prompt：

- 只允许写入当前 run 的 `prompt/` 目录
- 不能回写到 `assets/`
- 只能被当前 run 引用

## 7. 与 `video-copy-analyzer` 的强对齐点

- 保持“安装部署 -> 环境验证 -> 首次使用设置 -> 阶段式工作流”的主结构
- 保持用明确目标、执行步骤、确认成功条件来组织每个阶段
- 保持最终播报的模板化写法
- 保持“工具说明尽量服务于 Agent 执行”的写法，不写空泛背景介绍

## 8. 允许最小改动的点

- 新增标准 Prompt 与 run 专属 Prompt 的分流规则
